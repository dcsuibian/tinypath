# TinyPath

[English](README.md) | 中文

TinyPath 是一个面向 IoT 场景的极简 JSON 取值表达式语言，是 JSONPath 的严格子集。

## 背景

JSONPath 功能完整，但存在以下问题：

- **跨语言实现不一致**：各语言第三方库的行为存在差异，同一表达式在不同运行环境中可能返回不同结果，无法作为跨端数据协议的可靠基础
- **实现复杂度高**：完整的 JSONPath 规范特性众多，独立实现成本高，依赖第三方库则存在维护风险
- **过度设计**：IoT 数据取值场景仅需其中一小部分特性，完整规范对此类场景而言是负担

TinyPath 针对上述问题，在 JSONPath 的语法基础上进行严格裁剪，以更小的规范范围换取更高的跨语言一致性与实现可维护性。

## 设计目标

- **极简**：仅保留 IoT 取值场景所需的核心特性
- **一致**：严格的语法规则消除歧义，保证各语言实现行为一致
- **可移植**：规范足够简洁，任何语言的开发者均可依据规范独立实现

## 适用场景

从结构已知的 IoT 设备 MQTT 消息中，按路径提取叶子节点值（字符串、数字、布尔值、null）。

输入数据：

```json
{
  "sensors": [
    { "type": "temp", "id": "01", "value": 36.5 },
    { "type": "humi", "id": "01", "value": 65.2 }
  ]
}
```

表达式：

```tinypath
$["sensors"][?(@["type"]=="temp"&&@["id"]=="01")]["value"]
```

结果：`36.5`

**不适用于**：动态结构查询、聚合统计、非叶子节点提取。

## 语法概览

| 语法 | 说明 |
|------|------|
| `$` | 根节点，所有表达式必须以此开头 |
| `["field"]` | 对象字段访问 |
| `[n]` | 数组下标访问（非负整数） |
| `[?(@["key"]==value)]` | 数组过滤，返回第一个匹配元素 |
| `&&` | 过滤条件的逻辑与组合 |

完整语法规范见 [spec/SPEC.zh-CN.md](spec/SPEC.zh-CN.md)。
