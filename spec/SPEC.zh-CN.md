# TinyPath 语言规范

版本：`1.0.0`

## 概述

TinyPath 是一个用于从 JSON 数据中提取值的极简查询语言，是 JSONPath 的严格子集。

### 设计目标

- **极简**：只支持必要的特性，不支持任何可选特性
- **跨语言一致**：不同语言的实现尽量保证语义一致，但遵循各语言对 JSON 数值的默认解析行为
- **IoT友好**：专为已知结构的设备数据点位取值设计

### 适用场景

TinyPath 适用于从**结构已知**的 JSON 数据中按路径提取值，典型场景是 IoT 设备的 MQTT 消息点位映射。

TinyPath **不适用于**：

- 结构未知的动态 JSON 查询
- 需要聚合、统计、计算的场景
- 非 JSON 格式的数据（如二进制协议）

## 语法

### 根节点

表达式必须以 `$` 开头，表示 JSON 根对象。

```tinypath
$
```

### 字段访问

用 `["fieldName"]` 访问对象的某个字段。

```tinypath
$["fieldName"]
```

字段名规则：

- 使用双引号包裹
- 支持任意字符串，但不能为空字符串

### 数组下标

用 `[n]` 访问数组中指定位置的元素，`n` 为非负整数。

```tinypath
$["items"][0]
```

规则：

- 只支持非负整数（`0`、`1`、`2`...），不支持负数下标（如 `-1`）
- 下标越界返回 `null`，不抛异常

### 数组过滤

用 `[?(@["key"]==value)]` 在数组中筛选元素，支持用 `&&` 组合多个条件。

单条件：

```tinypath
$["arrayField"][?(@["key"]==value)]
```

多条件（`&&`）：

```tinypath
$["arrayField"][?(@["key1"]==value1&&@["key2"]==value2)]
```

规则：

- `@` 表示当前数组元素
- `key` 使用双引号包裹，不能为空字符串
- `value` 仅可以是以下类型：
  - 字符串（String）：使用双引号包裹，内容支持任意字符包括空格，如 `"hello world"`
  - 数字（Number）：不加引号，如 `42`、`3.14`
  - 布尔型（Boolean）：不加引号，只能是 `true` 或 `false`
  - null：不加引号，只能是 `null`
- `==` 两侧不允许有空格
- `&&` 两侧不允许有空格
- 多个条件全部满足才匹配（逻辑与）
- 不支持 `||`（逻辑或）
- 返回**第一个**匹配的元素
- 无匹配时返回 `null`
- 只支持 `==`，不支持 `>`、`<`、`!=` 等其他比较运算符

### 空格规则

- 表达式结构中任何位置均不允许出现空格
- 唯一例外：字符串类型的 `value` 内部允许空格，如 `[?(@["name"]=="hello world")]`

### 组合

字段访问、数组下标和数组过滤可以任意组合，从左到右依次执行。

```tinypath
$["sensors"][?(@["type"]=="temp"&&@["id"]=="01")]["readings"][0]["value"]
```

## 完整示例

表达式：

```tinypath
$["sensors"][?(@["type"]=="temp"&&@["id"]=="01")]["readings"][0]["value"]
```

数据：

```json
{
  "sensors": [
    {
      "type": "temp",
      "id": "01",
      "readings": [
        {"value": "36.500", "ts": 1700000001},
        {"value": "36.800", "ts": 1700000002}
      ]
    },
    {
      "type": "humi",
      "id": "01",
      "readings": [
        {"value": "65.200", "ts": 1700000001}
      ]
    }
  ]
}
```

结果：`"36.500"`

## 返回值

| 情况 | 返回值 |
| ---- | ------ |
| 正常找到 | 目标字段的原始值（字符串、数字、布尔、对象、数组均可） |
| 路径中某段不存在 | `null` |
| 数组过滤无匹配 | `null` |
| 数组下标越界 | `null` |
| 对非对象做字段访问 | `null` |
| 对非数组做过滤或下标访问 | `null` |
| 语法错误 | 抛出异常，不返回值 |

### 跨语言数值类型说明

TinyPath 各语言实现尽量保证语义一致，但数值类型遵循各语言对 JSON 的默认解析行为：

| 语言 | 整数 | 浮点数 |
|------|------|--------|
| Python | `int`（任意精度） | `float` |
| Java | `Long` / `Integer` | `Double` |
| JavaScript | `number` | `number` |

## 不支持的特性

以下 JSONPath 特性 TinyPath 明确不支持：

| 特性 | 示例 | 不支持原因 |
| ---- | ---- | ---------- |
| 通配符 | `$["sensors"][*]` | 点位名称是确定的 |
| 递归下降 | `$..["value"]` | 结构是确定的 |
| 负数下标 | `$["sensors"][-1]` | 不支持 |
| 数组切片 | `$["sensors"][0:3]` | IoT场景不需要 |
| 数值比较 | `[?(@["v"]>10)]` | 只需精确匹配 |
| 逻辑或 | `[?(@["a"]=="x"||@["b"]=="y")]` | 不支持 |
| 脚本表达式 | `[(@.length-1)]` | IoT场景不需要 |
| 单引号 | `$['sensors']` | 统一使用双引号 |
| 点号语法 | `$.sensors` | 统一使用括号语法 |
| 结构空格 | `[?(@["k"] == "v")]` | 字符串内部除外 |

## 错误处理

| 错误情况 | 行为 |
| -------- | ---- |
| 表达式不以 `$` 开头 | 抛出异常 |
| 包含不支持的语法 | 抛出异常 |
| 字段名为空字符串 | 抛出异常 |
| `==` 两侧有空格 | 抛出异常 |
| `&&` 两侧有空格 | 抛出异常 |
| 表达式中途截断 | 抛出异常 |
| 路径中间某段为 `null` | 返回 `null`，不抛异常 |
| 过滤后无匹配 | 返回 `null`，不抛异常 |
| 数组下标越界 | 返回 `null`，不抛异常 |

原则：**语法错误抛异常，数据问题返回 `null`**。

## 合规测试

所有 TinyPath 实现必须通过 `tests.json` 中定义的全部测试用例。

测试用例格式：

```json
{
  "description": "测试描述",
  "data": {},
  "expression": "$[...]",
  "expected": "期望结果或null"
}
```

新增实现时，直接读取 `tests.json` 运行测试即可验证合规性。
